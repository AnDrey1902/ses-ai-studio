# `<energy-monitor>` — live station-monitoring widget

A self-contained, embeddable Web Component that shows an animated "live" solar/backup-power
station: an hour-by-hour generation chart that plays through the day on a loop, switchable
object types (e.g. household vs. business) and station powers, and tariff-driven monthly
energy/savings figures.

- **Zero dependencies, no build step** — one `.js` file.
- **Fully isolated** — Shadow DOM. Drop many on a page; each has its own state, timer and styles.
  Nothing leaks in or out.
- **Configurable** — declaratively via attributes, or fully via a JS config object.
- **Themeable** — per-instance via CSS custom properties.

## Install

```html
<script src="widgets/energy-monitor/energy-monitor.js" defer></script>

<energy-monitor type="home" power="10"></energy-monitor>
```

That's it. The element renders its own card surface. Best with the **Sora** + **Manrope** fonts
loaded (it falls back to the system stack if they're absent).

## How it behaves

- Bars light up hour by hour: past hours go solid green, the **current hour glows amber** with its
  actual kW labelled above it, future hours stay ghosted (so you see the day's shape).
- The big number = current-hour generation; **сьогодні** accumulates live through the day.
- **цей місяць** / **заощаджено** are computed from the active station's power and its type tariff
  (using the current calendar month's measured per-kW output).
- A year row shows **вироблено на зараз** (cumulative Jan → selected month), **прогноз за рік**
  (forecast = all 12 months) and **заощаджено за рік** (yearly savings) — all tariff-driven.
- A compact **monthly breakdown** chart (months 1…12): every month up to the selected one is filled in
  the accent colour (the period that has "passed"), and the selected month is labelled with its output.
  In **auto** mode the selected month tracks today's date; with **АВТО off (manual)** the user can
  click / tap / hover any month to scrub — the cumulative «вироблено на зараз» updates to match.
- The **АВТО switch** sits under the ONLINE badge. Besides the switch, **any click inside the card**
  drops out of auto into manual mode (so a single tap on a month both pauses auto and selects it).
- Monthly/yearly numbers use **measured per-kW data**: `monthlyPerKw` is kWh generated by **1 kW** of
  capacity each month; every figure = `monthlyPerKw[m] × station kW`. Defaults are a real Ukrainian
  10 kW station ÷ 10 (≈ 1249 kWh/kW·year). Override per project via the `monthly-per-kw` attribute or
  `config`. Bar heights are derived from the same array.
- At the end of the day it auto-advances to the next station in the sequence and loops forever.
- The user can switch type/power manually at any time; auto-advance continues from there.
- An **АВТО on/off switch** (top-right): **on** = looping animation (current behaviour);
  **off** = freezes on the full day's end result so the user can read the totals and switch
  type/power manually. Maps to the `play()` / `pause()` API.
- Pauses itself while scrolled out of view, and respects `prefers-reduced-motion` (shows a static
  full-day state instead of animating).

## Configure with attributes

| Attribute | Default | Notes |
|---|---|---|
| `type` | `home` | initial object-type key |
| `power` | `10` | initial station power (kW) |
| `autoplay` | `true` | set `"false"` to freeze on a full-day snapshot |
| `hour-ms` | `650` | ms per simulated hour |
| `end-pause-ms` | `1200` | pause at day's end before switching station |
| `start-hour` | `6` | clock hour the profile begins at |
| `monthly-per-kw` | 12 measured values | kWh per **1 kW** per month (Jan…Dec); all totals scale by station kW |
| `tariff-home` / `tariff-biz` | `4.32` / `16` | price per kWh |
| `powers-home` / `powers-biz` | `5,10,30` / `30,50,100` | comma list of selectable powers |
| `label-home` / `label-biz` | `Дім` / `Бізнес` | type tab labels |
| `profile` | solar curve (13 pts) | comma list, normalized hourly generation, peak = 1 |
| `axis` | `6:00,12:00,18:00` | comma list of x-axis labels |
| `title` | `Моніторинг станції` | |
| `unit` / `energy-unit` / `energy-unit-big` | `кВт` / `кВт·год` / `МВт·год` | |
| `currency` / `locale` | `₴` / `uk-UA` | for number/money formatting |
| `online-label` / `now-label` / `today-label` / `month-label` / `saved-label` | UA defaults | |
| `month-labels` | `1,…,12` | comma list of 12 month labels |
| `current-month` | real month | `0`–`11` to pin the highlighted month |
| `months-title` | `Виробіток по місяцях` | heading above the monthly chart |
| `year-now-label` / `year-forecast-label` / `year-saved-label` | UA defaults | year-row labels |
| `auto-label` | `Авто` | label next to the on/off switch |

```html
<energy-monitor
  title="Станція №2"
  type="biz" power="50"
  powers-home="3,6" powers-biz="50,100,200"
  tariff-home="4.32" tariff-biz="18.5"
  hour-ms="420"></energy-monitor>
```

## Configure with a JS object (full control)

Set the `.config` property — it deep-merges over the attributes/defaults and re-renders.

```js
document.querySelector('energy-monitor').config = {
  startHour: 0,
  axis: ['0:00', '12:00', '24:00'],
  profile: [0,0,0,0,0,0,.05,.18,.4,.62,.8,.93,1,.98,.88,.72,.52,.3,.12,.02,0,0,0,0],
  types: {
    home: { label: 'Дім',    tariff: 4.32, powers: [5, 10, 30] },
    biz:  { label: 'Бізнес', tariff: 16,   powers: [30, 50, 100] }
  }
  // sequence: [...]   // optional; auto-derived from `types` when omitted
};
```

> **Swapping in a real curve:** replace `profile` (and `axis`/`start-hour`) with your measured
> 0:00–24:00 generation, normalized so the peak hour = `1`. Everything else (kW labels, kWh, ₴)
> scales off it automatically.

## Theme per instance (CSS custom properties)

These pierce the Shadow DOM, so set them on the element (inline or via a class):

```css
energy-monitor.teal {
  --em-accent: #2dd4bf;          /* active-hour / numbers */
  --em-accent-2: #0d9488;
  --em-grad-sun: linear-gradient(135deg, #5eead4, #0d9488);
  --em-surface: rgba(13,148,136,.12);
}
```

Other tokens: `--em-green`, `--em-green-2`, `--em-border`, `--em-blur`, `--em-text`, `--em-muted`,
`--em-muted-2`, `--em-radius`, `--em-pad`, `--em-shadow`, `--em-grad-green`,
`--em-font-display`, `--em-font-body`, `--em-live`.

**Size tokens** (to fit tight layouts — e.g. a hero that must stay within the viewport):
`--em-gap` (space between card sections), `--em-value-fs` (the big "current" number),
`--em-chart-h` (hourly-chart height), plus `--em-pad` / `--em-radius`. The site's hero sets compact
values on `.hero__monitor` so the whole card fits on laptops.

## JavaScript API

```js
const el = document.querySelector('energy-monitor');
el.setStation('biz', 100);  // switch programmatically
el.pause();                 // freeze on the day's end result (АВТО off)
el.play();                  // resume the looping animation (АВТО on)
el.config = { ... };        // re-configure (deep-merged)
```

## Visual builder + per-widget folder generator

[`demo.html`](demo.html) is a **visual configurator**: add / remove widgets, edit every setting
through a form, and see a live preview of each one. Settings persist to `localStorage`.

To turn configured widgets into **standalone, copy-anywhere folders**:

1. In `demo.html`, click **«💾 Зберегти configs.json»** and save the file next to this README
   (`widgets/energy-monitor/configs.json`).
2. Run the generator:

   ```bash
   node generate.mjs
   ```

3. You get `dist/<slug>/` per widget, each containing `index.html` + `energy-monitor.js`. Copy that
   whole folder onto any page or website — it runs on its own.

(Browsers can't write to disk from a page, so the form exports `configs.json` and the Node script
materialises the folders.) Prefer not to generate files? Each widget card also has
**«⧉ Скопіювати embed-код»**, which copies a ready `<energy-monitor>` snippet.

## Browser support

Any evergreen browser (Chrome/Edge/Firefox/Safari) — uses native Custom Elements + Shadow DOM.
The generator needs Node 16+.
