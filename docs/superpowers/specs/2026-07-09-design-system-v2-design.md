# Design System v2 — GURU ENERGY (ses-ai-studio)

> Spec / design doc. Статус: **утверждено направление, готово к плану внедрения.**
> Дата: 2026-07-09. Автор задачи: модернизация дизайн-системы активного проекта
> `ses-ai-studio` на основе лучшего из `LP_Guruenergy_ACC` + `ses-ai-studio`.

## 1. Контекст и цель

`ses-ai-studio` (React 19 + Tailwind v4 + Vite) и `LP_Guruenergy_ACC` (статический
vanilla-лендинг) — два воплощения одного бренда (GURU ENERGY / «Енергоживлення»).
LP визуально более зрелый (дисплейный шрифт, fluid-токены, дисциплина «секция ≤ 1
экран», bento), но это только лендинг. ses-ai-studio функционально богаче (AI-консультант,
калькулятор, магазин, блог, CRM, i18n×3), но дизайн-система «размазана» инлайн-классами.

**Цель:** взять лучшее из обоих и модернизировать дизайн-систему *активного* проекта
ses-ai-studio до цельной современной системы для сайта про **СЕС, ДБЖ и накопичувачі
енергії** (проєктування, монтаж, продаж).

### Установленные факты (doc ↔ реальность в ses-ai-studio)
- **Sora — фантом.** `docs/DESIGN-SYSTEM.md` заявляет Sora, но `index.html` грузит только
  Manrope + Inter; в коде `Sora` не встречается. Дисплейного шрифта фактически нет.
- **Токены `@theme` почти не работают.** `src/index.css` объявляет `--color-*`, но компоненты
  хардкодят хекс инлайном (`text-[#1A2E23]`, `bg-[#F8FAF9]`, `bg-[rgba(34,197,94,.03)]`).
- **«Полосатость».** Посекционное dark→light→dark без единой системы поверхностей.
- **Нет шкал** радиусов/теней (россыпь 8/12/14/16/28/32 + инлайн box-shadow).
- **energy-monitor виджет уже на целевой палитре** (`#22b865/#0f7a40/#ffc742/#f59e0b/#9fbcae/#04130b`
  = токены LP). Значит окружающий UI сейчас *рассинхронизирован* с монитором; v2 это чинит.

## 2. Рамки (HARD constraints)

**2.1. Только презентационный слой.** Меняем: токены, стили, CSS, макет/визуал.
**НЕ трогаем:** функционал, логику компонентов, data-flow, состояние (AppContext,
localStorage), i18n-ключи и тексты, состав/порядок секций и блоков, виджеты
(`energy-monitor`, AI-консультант), калькулятор, магазин, CRM/лиды, роутинг.
DOM-структура компонентов сохраняется; правим классы/стили, а не архитектуру.

**2.2. «Секция ≤ 1 экран».** Контент каждой основной секции должен помещаться в один
экран ноутбука (~768px окно ≈ 650–700px полезных) — без прокрутки *внутри* секции.
Реализация: fluid-rem отступы с низким floor, vh-aware clamps на gaps/margins, медиа
кэпится через vh (`height: clamp(280px, 48vh, 520px); object-fit: cover`), не через
высокий фиксированный `aspect-ratio`. **Критерий проверки:** на 1366×768 и 1440×900
верхний заголовок И нижний CTA секции видны без скролла.

**2.3. energy-monitor не трогаем** (ни поведение, ни цвета — он уже на v2-палитре).
Монитор остаётся центром hero как есть.

## 3. Токен-система (источник правды)

Всё живёт в `src/index.css` `@theme` (Tailwind v4, CSS-first). Компоненты используют
семантические Tailwind-утилиты (`bg-surface`, `text-ink`, `text-emerald` и т.п.) и
тонкий слой `.ds-*` классов — **хардкод хекса инлайном устраняется**.

### 3.1. Цвет (primitives)
| Токен | Hex | Роль |
|---|---|---|
| `--color-ink` | `#0a2a1e` | Глубокий лес — тёмный ink / база dark-секций |
| `--color-ink-2` | `#0d3527` | Слой ink |
| `--color-text` | `#16271f` | Body/заголовки на светлом (~14:1 на белом) |
| `--color-muted` | `#54665d` | Вторичный текст на светлом (~5.4:1) |
| `--color-muted-dark` | `#9fbcae` | Вторичный текст на тёмном |
| `--color-surface` | `#ffffff` | Страница / карточки |
| `--color-soft` | `#eef5f0` | Мягкая чередующаяся секция |
| `--color-line` | `#dde8e2` | Границы / разделители |
| `--color-emerald` | `#18a558` | **Основной CTA / интерактив / ссылки** |
| `--color-emerald-deep` | `#0f7a40` | Hover / обводки / текст-акцент на светлом |
| `--color-sun` | `#ffc742` | Золото: ключевые слова, цифры, бейджи |
| `--color-amber` | `#f59e0b` | Вторичный акцент / «внимание» |

### 3.2. Градиенты
```css
--grad-emerald: linear-gradient(135deg, #22b865 0%, #0f7a40 100%);  /* primary CTA */
--grad-sun:     linear-gradient(135deg, #ffd23f 0%, #f59e0b 100%);  /* solar emphasis */
--grad-dark:    linear-gradient(160deg, #11402f 0%, #0a2a1e 72%);   /* dark sections / hero */
```

### 3.3. Контрастные правила (не регрессировать)
- Светлые поверхности: текст `--color-text`/`--color-ink`; muted `--color-muted`.
- Dark-секции: текст `#fff`; muted `--color-muted-dark` / `#bcd2c8`.
- Emerald CTA: текст near-black `#04130b`. Sun/amber: тёмный текст (`#2a1a00`).
- **Никогда** светло-серый текст на светлом фоне.

### 3.4. Типографика
```css
--font-display: "Sora", system-ui, sans-serif;      /* 600/700/800 — заголовки, цифры */
--font-body:    "Manrope", system-ui, sans-serif;   /* 400–800 — текст, UI */
--font-mono:    "JetBrains Mono", ui-monospace, monospace;  /* только служебно: токены/таблицы */
```
- **Реально подключить Sora** в `index.html` Google Fonts (закрыть «фантом»). Inter удаляется
  из загрузки, если не используется (проверить перед удалением).
- Дисплей ≤ 6rem, letter-spacing −0.02em, `text-wrap: balance` на h1–h3.
- Длина строки body 65–75ch. Ключевые слова/цифры подсвечиваются `--color-sun`.
- Fluid заголовки: `--title-fs: clamp(1.55rem, 1.4vw + 1.6vh + .6rem, 2.7rem)`.

### 3.5. Шкалы
- **Радиус:** `--r-sm 12px · --r-md 18px · --r-lg 26px · --r-pill 999px` (карточки по умолч. `--r-md`).
- **Отступы (fluid rem):** `--gap: clamp(1rem,3vw,2rem)`,
  `--section-pad: clamp(1.8rem, 2.6vw + 2.8vh, 5rem)`, container `--maxw: 1200px`.
- **Тени:** `--shadow-sm 0 2px 10px rgba(7,20,15,.06)`, `--shadow-md 0 14px 40px rgba(7,20,15,.10)`,
  `--shadow-lg 0 30px 70px rgba(7,20,15,.16)`, `--shadow-emerald 0 12px 30px rgba(24,165,88,.28)`.

## 4. Тема

**Light-first** (как LP_Guruenergy). Светлые поверхности по умолчанию; dark-секции —
осознанный контраст через `--grad-dark` + белый текст, а не случайная «полосатость».
Вводится **единая система поверхностей**: `surface` (белые карточки) / `soft` (мягкая
чередующаяся секция) / `dark` (градиентная контрастная секция). Каждая секция явно
принадлежит одному из трёх типов. Глобального light/dark-toggle нет (вне рамок).

## 5. Компоненты (`.ds-*` + утилиты)

Перенос вида в CSS-классы (по образцу LP `css/styles.css`), чтобы компоненты не
хардкодили стили:
- **Кнопки:** `.ds-btn-primary` (`--grad-emerald` + `#04130b`), `.ds-btn-outline`
  (изумрудная обводка), `.ds-btn-ghost` (`currentColor`), `.ds-btn-sun` (`--grad-sun`).
- **Карточка:** `.ds-card` — `surface`, `1px solid --color-line`, `--r-md`, `--shadow-sm/md`.
- **Секция:** типы `--section-soft` / `--section-dark`; отступы из `--section-pad`.
- **Бейдж:** `.ds-badge` варианты green / sun / amber.
- **Поле:** `.ds-input` — `--r-md`, focus-ring изумруд `0 0 0 3px rgba(24,165,88,.18)`.
- **Bento:** сетка услуг (feature-карточка + мелкие), паттерн из LP.
- **Фокус:** видимый `:focus-visible` outline изумруд, 2px, offset 2px (уже есть — сохранить).
- **Motion:** ease-out, без bounce; `prefers-reduced-motion` обязателен (уже есть — сохранить).

## 6. Сигнатура (energy-flow) — декоративно

Мотив «поток энергии» (солнце → поток → дом с панелью/накопителем) применяется
**только на уровне стилей/декора** (фоновая графика/акцент в dark-секциях), **без**
нового структурного блока и **без** конкуренции с energy-monitor. Опционально; если
усложняет — опустить. Boldness тратим в одном месте.

## 7. Объём внедрения (scope)

**Система + флагманские секции.**
1. **Токены:** переписать `src/index.css` `@theme` под раздел 3 (+ `.ds-*` классы, утилиты).
2. **Шрифты:** подключить Sora в `index.html`; проверить/убрать неиспользуемый Inter.
3. **Документация:** переписать `docs/DESIGN-SYSTEM.md` под реальность v2 (single source).
4. **Рефактор-эталон (presentation-only, DOM/логика/i18n/виджеты сохраняются):**
   - `HeroSection.tsx` — фон/подача под light-first + dark-hero, монитор как есть.
   - `ServicesSection.tsx` — bento на новые токены/классы, убрать инлайн-хекс.
   - `PricesSection.tsx` — карточки/бейджи на `.ds-*`.
   - `ContactFormSection.tsx` — форма на `.ds-input`/`.ds-btn`.

Остальные ~9 секций и страницы (магазин, блог, админ) переводятся по образцу **позже**,
вне этого захода.

## 8. Не в объёме (out of scope)

Функционал, тексты/контент, порядок и состав секций, поведение виджетов/монитора/AI/
калькулятора/магазина/CRM, i18n-ключи, роутинг, серверная часть, глобальный theme-toggle,
рефактор нефлагманских секций и внутренних страниц.

## 9. Критерии приёмки

- `npm run lint` (tsc --noEmit) проходит; `npm run build` собирается.
- Флагманские секции визуально соответствуют v2 (палитра, шрифты, шкалы) и **не содержат
  инлайн-хекса** там, где есть токен.
- Sora реально загружается и применяется к заголовкам.
- **Секция ≤ 1 экран:** на 1366×768 и 1440×900 у флагманских секций верхний заголовок и
  нижний CTA видны без скролла внутри секции.
- Контраст: body ≥ 4.5:1, крупный текст ≥ 3:1 на всех фонах.
- **Ноль функциональных регрессий:** лиды, AI-консультант, калькулятор, монитор, навигация,
  i18n, модалка — работают как до правок. DOM-структура и порядок секций не изменились.
- `prefers-reduced-motion` и видимый keyboard-focus сохранены.

## 10. Верификация

Прогнать `npm run dev`, открыть флагманские секции на 1366×768 и 1440×900 (chrome-devtools),
проверить: fit-в-экран, контраст, загрузку Sora, отсутствие визуальных багов, работу
монитора/AI/форм. Скриншот-самокритика до «готово».
