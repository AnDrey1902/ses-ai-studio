# СЕС під ключ — Технічна специфікація лендингу
**Версія:** 1.2 (фінальна)  
**Дата:** 26.06.2026  
**Тип:** Lead-generation Landing + Blog + Shop (заглушка) + Admin  
**Стек:** Next.js + Payload CMS  
**Мови:** UA / RU / EN  

---

## 1. Огляд проекту

**Мета:** Генерація кваліфікованих лідів для компанії з монтажу СЕС (сонячних електростанцій) в Україні через інформаційно насичений лендинг з блогом.

**Основна дія користувача:** Залишити заявку (ім'я + телефон) або зателефонувати.

**Первинна мета:** Генерувати вхідні заявки / дзвінки.

**Вторинна мета:** Будувати довіру та SEO-трафік через блог і контент.

**Метрика успіху:** Кількість заявок на тиждень; конверсія лендингу (ціль — 2,5–4% відвідувачів у заявку).

---

## 2. Цільова аудиторія

**Основний користувач (B2C):** Власник приватного будинку 30–55 років. Шукає захист від блекаутів або заробіток на зеленому тарифі. Пошукові запити: "СЕС під ключ 10 кВт", "гібридна сонячна електростанція", "кредит 0% на сонячні панелі".

**Другорядний користувач (B2B):** Власник або директор МСБ (магазин, склад, АЗС, ферма). Шукає зниження рахунків за електрику та захист від блекаутів.

**Контент-менеджер / адміністратор:** Нетехнічний співробітник компанії, редагує весь контент сайту через Payload CMS адмінку.

**Точка входу:** Google Search (основна), Facebook/Instagram реклама (вторинна), Viber/Telegram-реферал.

## 3. Технічний стек

### 3.1. Двоетапний деплой

**Етап 1 — Preview (Vercel + Neon):** Швидкий старт для погодження дизайну та функціоналу. Безкоштовно.

**Етап 2 — Production (власний VPS + Coolify):** Переїзд після погодження. Просто міняємо `DATABASE_URI` та `PAYLOAD_SECRET` у `.env`. Код не змінюється.

| | Etap 1: Vercel | Etap 2: VPS |
|---|---|---|
| Хостинг Next.js | Vercel (Hobby, free) | Hetzner CX22 + Coolify |
| База даних | Neon PostgreSQL (free, 0.5 GB) | PostgreSQL у Docker |
| Медіа | Локальна папка `/public/uploads` | Локальна папка `/uploads` (persistent) |
| Домен | `.vercel.app` або свій | Свій домен |
| SSL | Vercel автоматично | Coolify + Let's Encrypt |
| Вартість/міс | $0 | €4.5 (~$5) |

> ⚠️ **Важливо про медіа на Vercel:** Vercel serverless не має persistent disk. Файли, завантажені через адмінку, скидаються при наступному деплої. Для Preview-етапу це прийнятно (контент тестовий). На Production-VPS Payload зберігає медіа у звичайну папку `/uploads` — persistent storage за замовчуванням.

## 4. Мультимовність

**Мови:** Українська (UA, дефолт) · Російська (RU) · Англійська (EN)

**Реалізація:** Payload CMS 3.x — вбудована локалізація (`localization: { locales: ['uk', 'ru', 'en'], defaultLocale: 'uk' }`). Всі текстові поля колекцій мають переклади. Перемикач мови — у хедері сайту.

**URL-структура:**
```
/              → UA (дефолт)
/ru/           → RU
/en/           → EN
/blog          → UA
/ru/blog       → RU
/en/blog       → EN
```

**Next.js реалізація:** `next-intl` або вбудований `i18n` роутинг. `generateStaticParams` з параметром `locale`.

**Що перекладається (через адмінку):**
- Всі тексти блоків лендингу (через `site-settings` global)
- Статті блогу (окремі переклади)
- Кейси, відгуки, FAQ
- SEO-метадані (title, description, og) — окремо для кожної мови

**Що НЕ перекладається (статично в коді):**
- UI елементи (кнопки "Submit", лейбли форм) — через JSON-файли локалізації
- Системні повідомлення форми

## 7. Функції та бізнес-логіка

### 7.1. Форма заявки [v1]
1. Клієнт заповнює форму (name, phone, power?, sesType?)
2. Frontend: валідація (phone mask, name ≥ 2 символи)
3. `POST /api/leads` — Next.js Route Handler
4. Honeypot перевірка → якщо заповнений hidden field → reject 200 (без помилки, щоб не підказувати ботам)
5. Rate limit: 5 запитів / 10 хв з IP (in-memory Map або Upstash Redis)
6. Payload: `leads.create()` — запис у БД з полем `lang` (поточна мова)
7. Resend: email адміну "🔔 Нова заявка: [name], [phone], [power] кВт, [sesType]"
8. Frontend: success-стан

**Scope:** [v1]

---

### 7.2. Мультимовний контент [v1]
- Payload localization повертає контент поточної локалі
- Next.js `[locale]` segment у роуті
- `next-intl` для UI-рядків (кнопки, лейбли, системні повідомлення)
- `generateMetadata` генерує `hreflang` теги для SEO

**Scope:** [v1]

---

### 7.3. Завантаження медіа [v1]
- Payload Media collection: upload через адмінку
- Файли зберігаються: `./uploads/` (VPS, persistent) або `./public/uploads/` (Vercel, тимчасово)
- Payload автоматично генерує thumbnail 300×300 та medium 800×600
- У Next.js: `next/image` з `unoptimized={false}`, домен додається до `next.config.js`

**Scope:** [v1]

---

### 7.4. Блог + SEO [v1]
- ISR: `revalidate: 3600` (перегенерація кожну годину)
- `generateMetadata` з Payload seo-полів
- OG image: статичне з `/uploads` або авто-генерація `/api/og` через `@vercel/og`
- Sitemap: `app/sitemap.ts` — динамічна генерація з усіх `posts` + статичних сторінок × локалі

**Scope:** [v1]

---

### 7.5. Калькулятор окупності [v1]
- Повністю client-side JS, без API
- Формула:
  ```
  виробіток = кВт × 1100 (кВт·год/рік, середнє по UA)
  тариф_ЗТ  = 0.132 € (константа 2025, оновлюється в коді)
  дохід_нетто = виробіток × тариф_ЗТ × 0.805
  окупність   = ціна_$ / (дохід_нетто / курс_EUR_USD)
  ```

**Scope:** [v1]
